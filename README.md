# 🚀 SocialSpread Workflow (社交媒体全自动分发系统)

## 📝 简介 (Introduction)
SocialSpread Workflow 是一个基于 Python 的全自动化社交媒体内容分发系统。它旨在解决内容创作者在多平台发布时的痛点：繁琐的排版调整、不同平台的风格适配以及重复的上传操作。

通过结合 Playwright 自动化技术和 AI 大模型（如豆包 AI），本系统能够将一篇 Markdown 格式的原始文章，自动改写为适配微信公众号、小红书等不同平台风格的内容，并自动完成发布或推送到草稿箱。

## 🎯 适用人群与场景
**适用人群**：
- 技术博客作者、自媒体运营者
- 需要维护企业技术博客的开发者
- 希望通过自动化节省分发时间的个人开发者

**使用场景**：
- **一次编写，多处运行**：写好一篇 Markdown 技术文章，自动分发到公众号（适配排版）、小红书（生成长图文风格）等平台。
- **AI 风格化重写**：针对小红书需要更活泼、带 Emoji 的风格，针对公众号需要更正式的排版，系统自动根据预设 Prompt 进行改写。
- **自动化发布**：对于提供 API 的平台（如微信公众号）直接调用接口；对于无 API 的平台（如小红书），使用浏览器自动化模拟人工操作。

## ✨ 核心特性 (Features)

### 🤖 多模态自动化引擎
- **API 集成**：原生支持微信公众号（草稿箱）接口。
- **RPA 模拟**：内置 Playwright 脚本，支持小红书网页版的全自动登录（需扫码）、内容填写、标签选择和发布。

### 🧠 AI 智能改写与生成
- **三维提示词工程**：正文改写提示词由“用户个性风格 + 平台风格 + 本次发布特殊要求”三部分组成，精准控制输出效果。
- **多场景支持**：支持针对正文改写和封面图生成的独立特殊提示词输入。

### 🛡️ 智能缓存与断点续传
- **内容复用机制**：AI 生成的摘要、改写正文、Tag 和封面图会自动保存在源文件同级目录下（如 `xhs正文.txt`, `weixin正文.txt`）。
- **优先读取缓存**：再次运行时，系统会自动检测并优先使用本地已存在的 AI 处理结果，避免重复消耗 Token 和时间。

### 🏗️ 自愈式交互架构 (New!)
- **场景工作流**：基于“交互流程链 + 原子交互节点”构建，每个节点包含页面前置条件、动作类型、录制结果和验证规则。
- **自动录制修复**：执行过程中若遇到元素交互失败，系统会自动触发录制脚本重新捕获该原子节点的交互元素，并自动重试流程。
- **模块化设计**：解耦流程与元素定位，适应网站改版，未来支持 AI 自动探索和维护流程。

## 🚀 极简部署指南 (How to Use)

### 阶段一：环境准备
确保你的电脑已安装 Python 3.8+ 和 Chrome 浏览器。

1.  **克隆项目并安装依赖**：
    ```bash
    # 推荐使用 uv 管理依赖，或者直接 pip
    pip install -r requirements.txt
    # 安装 Playwright 浏览器驱动
    playwright install chromium
    ```

2.  **配置环境变量**：
    - 复制 `.env.example` 为 `.env`。
    - 填入你的微信公众号 `WECHAT_APP_ID`, `WECHAT_APP_SECRET`。
    - 根据需要调整 AI 提示词配置。
    
### 阶段二：运行发布
使用 `pytest` 命令启动自动化流程。

**示例 1：发布到所有平台（基础用法）**
```bash
pytest -s test_social_media_automatic_publish.py --title="我的新文章" --url="https://example.com" --markdown-file="./articles/my_article.md" --platforms="all"
```

**示例 2：仅发布到小红书并开启 AI 改写（进阶用法）**
```bash
pytest -s test_social_media_automatic_publish.py \
  --title="我的新文章" \
  --url="https://example.com" \
  --markdown-file="./articles/my_article.md" \
  --platforms="xiaohongshu" \
  --rewrite-platform-content="true" \
  --rewrite-special-prompt="尽量保持幽默风趣，多用emoji" \
  --image-special-prompt="赛博朋克风格，霓虹灯效果"
```

**示例 3：指定 Markdown 文件路径**
```bash
pytest -s test_social_media_automatic_publish.py --title="深度学习入门" --url="https://example.com" --markdown-file="./articles/deep_learning.md"
```

## 📅 最近更新 (Latest Updates)

### 2026-03-07: 架构升级与自愈机制
- **架构改变**：引入“场景工作流 + 录制机制”架构。
    - 工作流由交互流程链和原子交互节点（前置条件、动作、结果、验证）构成。
    - **自愈能力**：当自动化交互中某个元素点击失败时，自动调用录制脚本重新录制该原子节点，更新选择器后自动重试。
    - **优势**：模块化、解耦，大幅降低因网站 UI 变动导致的维护成本。

### 2026-03-05: 提示词增强与缓存机制
- **新需求**：正文改写提示词升级为三段式结构（用户风格 + 平台风格 + 特殊要求）。
    - 支持通过命令行参数 `--rewrite-special-prompt` 和 `--image-special-prompt` 传入本次发布的即时需求。
- **新需求**：增加 AI 生成内容的本地缓存功能。
    - AI 处理后的正文、摘要、封面图按平台分类保存（如 `xhs正文`、`weixin正文`）。
    - 运行时优先检查本地缓存，存在则直接使用，提高效率并节省成本。
