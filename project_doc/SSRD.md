# 软件系统需求定义 (SSRD)

## 1. 上层抽象需求
- **自动化社交媒体发布**：构建一个能够自动处理和发布内容到社交媒体平台（主要是微信公众号）的系统。
- **AI内容辅助**：利用AI（豆包）进行文章内容的改写、摘要生成和封面图片生成。
- **多平台支持**：虽然目前侧重于微信公众号，但架构上应支持多平台扩展（如知乎、CSDN等，目前代码中有相关迹象）。
- **稳定性与鲁棒性**：系统应能应对第三方平台（如豆包Web端）的UI变化，保持核心功能的稳定性。
- **自愈式交互自动化**：针对第三方平台 UI 频繁变动的问题，建立“交互原子化 + 失败自动重录”机制，将维护成本分摊到运行时，而非代码重构。

## 2. 行业案例调研
- **类似工具**：
    - [Selenium/Playwright Automation Scripts]：GitHub上有许多基于Selenium或Playwright的自动化脚本，用于各种平台的签到、发布等。
    - [WeChat Official Account API Wrappers]：如 `wechatpy` 等库，封装了微信公众号API。
- **思考**：
    - 大多数自动化工具面临的最大挑战是平台反爬虫策略和UI频繁更新。
    - 使用Playwright的MCP（Model Context Protocol）或结合AI视觉识别元素（如本次修复中使用的SVG路径匹配或隐藏输入框上传）是解决UI变化的一种有效途径。
    - 官方API（如微信公众号API）比Web自动化更稳定，应优先使用API（如 `wechat_mp_sdk`）。
    - **新思路**：借鉴 RPA 工具（如 UiPath）的“录制-回放”逻辑，但将其原子化，只在出错时触发局部录制，结合 Playwright Codegen 实现轻量级维护。
