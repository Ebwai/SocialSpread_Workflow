# 需求规格说明书 (SRS)

## 1. 功能需求

### 1.1 内容生成与处理
- **文章改写**：
    - 输入：Markdown文章原文。
    - 过程：调用豆包AI（Web端）进行文章风格改写。
    - 输出：改写后的Markdown文章。
- **摘要生成**：
    - 输入：文章原文。
    - 过程：调用豆包AI生成文章摘要。
    - 输出：摘要文本。
- **封面生成**：
    - 输入：文章摘要文本。
    - 过程：调用豆包AI（Web端），基于摘要文本生成提示词，再生成图片。
    - 输出：生成的图片文件（PNG/JPG）。

### 1.2 社交媒体发布
- **微信公众号发布**：
    - 输入：Markdown文章、生成的封面图。
    - 过程：
        - 上传封面图片到微信公众号永久素材库（通过API）。
        - 上传文章内容到微信公众号（通过API或Web自动化）。
        - 获取文章的Media ID或发布链接。
    - 输出：已发布或草稿状态的公众号文章。

### 1.3 交互自愈与录制（增强）
- **单交互录制与回放**：
    - 需求：将复杂流程拆解为“原子交互”（如点击上传、输入文本），每个交互独立管理。
    - 失败处理：当某个交互因 Selector 失效而失败时，系统自动暂停。
    - 自动重录：
        1. 执行“最小前置链”以复现交互前的页面状态。
        2. 启动 Playwright Codegen 录制器。
        3. 用户操作一次目标元素并关闭窗口。
        4. 系统解析新 Selector 并更新 `atom_storage.json`。
        5. 自动重试该交互。
    - 状态：[✓] 已实现核心框架 `InteractionManager` 并接入豆包/小红书流程。

## 2. 非功能需求
- **稳定性**：能够应对豆包Web端UI的小幅度变化（如按钮SVG路径变更）。
- **兼容性**：支持主流操作系统（Windows/Linux/macOS）下的Playwright运行。
- **配置灵活性**：通过环境变量（.env）配置API Key、AppID等敏感信息。
- **可测试性**：提供完整的单元测试和集成测试覆盖核心流程（如 `tests/test_wechat_mp_workflow.py`）。
